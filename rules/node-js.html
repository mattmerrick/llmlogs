About These Rules

These Node.js coding rules are specifically formatted for the Cursor Editor as .mdc files, focusing on modern asynchronous patterns and error handling. Each rule is designed to help you write more robust Node.js applications while leveraging Cursor's AI capabilities for enhanced development.

1. Error Handling with Async/Await

Implement comprehensive error handling in async/await patterns using try-catch blocks and custom error classes. This approach ensures better error tracking, debugging, and maintenance of Node.js applications, especially in Express.js middleware chains.

nodejs-error-handling.mdc 
// src/errors/AppError.ts
export class AppError extends Error {
  constructor(
    public statusCode: number,
    message: string,
    public code: string,
    public isOperational = true
  ) {
    super(message);
    Object.setPrototypeOf(this, AppError.prototype);
  }
}

// src/middleware/errorHandler.ts
import { Request, Response, NextFunction } from 'express';
import { AppError } from '../errors/AppError';

export const errorHandler = (
  err: Error | AppError,
  req: Request,
  res: Response,
  next: NextFunction
) => {
  if (err instanceof AppError) {
    return res.status(err.statusCode).json({
      status: 'error',
      code: err.code,
      message: err.message
    });
  }

  // Unexpected errors
  console.error('Unexpected error:', err);
  return res.status(500).json({
    status: 'error',
    code: 'INTERNAL_ERROR',
    message: 'An unexpected error occurred'
  });
};

// src/controllers/user.ts
import { Request, Response, NextFunction } from 'express';
import { AppError } from '../errors/AppError';
import { UserService } from '../services/user';

export class UserController {
  constructor(private userService: UserService) {}

  createUser = async (req: Request, res: Response, next: NextFunction) => {
    try {
      const { email, password } = req.body;

      if (!email || !password) {
        throw new AppError(400, 'Missing required fields', 'INVALID_INPUT');
      }

      const user = await this.userService.create({ email, password });
      
      res.status(201).json({
        status: 'success',
        data: { user }
      });
    } catch (error) {
      // Let the error handler middleware handle it
      next(error);
    }
  };

  // Example of using async/await with proper error handling
  getUserById = async (req: Request, res: Response, next: NextFunction) => {
    try {
      const user = await this.userService.findById(req.params.id);
      
      if (!user) {
        throw new AppError(404, 'User not found', 'USER_NOT_FOUND');
      }

      res.json({
        status: 'success',
        data: { user }
      });
    } catch (error) {
      next(error);
    }
  };
}

Why This Rule Matters

  • Provides consistent error handling across the application
  • Improves debugging with detailed error information
  • Separates operational from programming errors
  • Ensures proper error responses in REST APIs
  • Makes error tracking and logging more effective

  • Suggest appropriate error types and status codes
  • Implement consistent error patterns across your codebase
